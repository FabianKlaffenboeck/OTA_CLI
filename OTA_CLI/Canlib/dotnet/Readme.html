Kvaser C# Wrapper for CANlib

The Kvaser C# wrapper for CANlib exists in two different versions, one is targeting .NET Standard 2.0 and the other is targeting .NET 6.0. This document describes the differences between the two versions and when to use which.

Using NuGet Package Manager

In this folder you will find NuGet package files, one for each wrapper. These packages can be used in your project using the NuGet Package Manager. To do this you need to add this folder as a NuGet package source for your project by adding a nuget.config file. Here is an example of a nuget.config file:

<?xml version="1.0" encoding="utf-8"?>
<configuration>
    <packageSources>
        <clear />    
        <add key="nuget.org" value="https://api.nuget.org/v3/index.json" protocolVersion="3" />
        <add key="canlib" value="[THE PATH TO THIS FOLDER EITHER ABSOLUT OR RELATIVE TO YOUR PROJECT]" />
    </packageSources>
</configuration>

If you are using this approach then the NuGet Package Manager will make sure that you are using the correct version of the wrapper, .NET Standard 2.0 or .NET 6.0.

Follow these steps to use the NuGet Package Manager:

  1. Add a nuget.config file to your projects root folder
  2. Make sure that the canlib source is pointing to the folder that includes the CANlib NuGet package files
  3. Open you project in Visual Studio
  4. Open the Solution Explorer
  5. Locate the Dependencies folder underneath your project, right-click on the folder and select 'Manage NuGet Packages'
  6. Click on Browse
  7. Make sure the 'Package source' is set to 'All' or 'canlib'
  8. Select the wrapper you want to use in your project and click Install

Using Assembly File References

Another alternative is to add the Kvaser C# wrapper assemblies as project references. In that case you must make sure to pick the correct version. These assemblies are found in sub-folders net6.0 respectively netstandard2.0.

If your project is targeting .NET 6.0 or higher, then you should reference the assemblies in sub-folder net6.0. This is the preferred version to use. The net6.0 assemblies include support for both Windows and Linux whilst the netstandard2.0 assemblies only supports Windows.

If your project is targeting any version below .NET 6.0, i.e. .NET Framework 4.x, .NET Core 2.x-3.x or .NET 5.x, then you should reference the assemblies in sub-folder netstandard2.0.

Follow these steps to use the file references:

  1. Open you project in Visual Studio
  2. Open the Solution Explorer
  3. Locate the Dependencies folder below your project, right-click on the folder and select 'Add Project Reference'
  4. Click on Browse in the left tree view and then on the Browse button
  5. Browse to the correct folder, net6.0 or netstandard2.0, select the assembly you want to use and click Add
  6. Click on OK
